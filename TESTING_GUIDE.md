# 🧪 دليل اختبار الصلاحيات

## الخطوات السريعة للاختبار

### 1. تشغيل النظام
```bash
npm run dev
# ثم افتح http://localhost:5173
```

### 2. اختبار رتب مختلفة

#### 🔴 **اختبر كـ مدير عام**
- **بيانات الدخول:** admin / 01147497465
- **ما يجب أن تراه:**
  ✅ جميع الصفحات (21 صفحة)
  ✅ جميع الأنظمة الجديدة
  ✅ كل الأزرار والمميزات

#### 💰 **اختبر كـ محاسب** (إذا كان موجوداً)
```
ما يجب أن يرى:
✅ الفواتير والعروض
✅ المحاسبة والضرائب
✅ الرواتب والعقود
✅ التقارير

ما لا يجب أن يرى:
❌ المشاريع (مباشرة)
❌ المخزون
❌ الإجازات
```

#### 👷 **اختبر كـ عامل** (إذا كان موجوداً)
```
ما يجب أن يرى:
✅ لوحة التحكم
✅ الحضور والغياب فقط
✅ البريد الإلكتروني
✅ غرفة الاجتماعات

ما لا يجب أن يرى:
❌ أي نظام آخر
❌ بيانات موظفين آخرين
❌ المشاريع أو الفواتير
```

### 3. اختبار الميزات

#### اختبر إنشاء فاتورة
```
1. سجل دخول كمدير عام
2. انقر على "الفواتير والعروض"
3. انقر على "فاتورة جديدة"
4. ملء البيانات والحفظ
✅ يجب أن تعمل بدون مشاكل
```

#### اختبر إنشاء مشروع
```
1. سجل دخول كمدير قسم
2. انقر على "إدارة المشاريع"
3. انقر على "مشروع جديد"
4. ملء البيانات والحفظ
✅ يجب أن تعمل
```

#### اختبر تسجيل الحضور
```
1. سجل دخول كعامل
2. انقر على "الحضور والغياب"
3. انقر على "تسجيل الحضور"
4. اختر "حاضر" والحفظ
✅ يجب أن تعمل
```

### 4. اختبار منع الوصول

#### محاولة الدخول غير المصرح
```
1. سجل دخول كموظف استقبال
2. حاول الدخول للرابط: /inventory
3. أو جرب الرابط: /accounting

❌ يجب أن ترى رسالة خطأ أو تحويل للرئيسية
```

## نقاط الاختبار الرئيسية

### ✓ القائمة الجانبية
- [ ] مدير عام يرى كل الخيارات
- [ ] محاسب يرى فواتير وحسابات فقط
- [ ] عامل يرى حضور وبريد فقط
- [ ] الخيارات تتغير حسب الرتبة

### ✓ الصفحات
- [ ] كل موظف يرى صفحاته فقط
- [ ] الصفحات الممنوعة تعطي خطأ
- [ ] الرجوع للرئيسية من الخطأ يعمل

### ✓ الأزرار والمميزات
- [ ] زر "إنشاء" يظهر فقط للمصرحين
- [ ] زر "حذف" يظهر فقط للمدراء
- [ ] زر "تعديل" متاح حسب الصلاحية

### ✓ البيانات
- [ ] كل موظف يرى بيانات منطقية لرتبته
- [ ] العاملون لا يرون رواتب الآخرين
- [ ] المحاسبون يرون كل الرواتب

## حالات الاختبار المرضية

### 1. إنشاء موظف جديد برتبة مختلفة
```javascript
// جرّب إضافة موظف جديد بكل رتبة:
- مدير عام
- محاسب
- عامل
- فني
- مهندس

// ثم سجل دخول بحسابهم واختبر الصلاحيات
```

### 2. تغيير رتبة الموظف
```javascript
// غيّر رتبة موظف من عامل إلى مهندس
// يجب أن تتحديث الصلاحيات فوراً
// عند سجول الدخول التالي
```

### 3. تسجيل أنشطة مختلفة
```javascript
// تأكد من تسجيل:
✓ محاسب يضيف فاتورة → يظهر في الإشعارات
✓ مدير يوافق على إجازة → يظهر النشاط
✓ فني يحدّث المخزون → يظهر في السجل
```

## النتائج المتوقعة

### ✅ اختبار ناجح إذا:
- كل رتبة ترى صفحاتها الصحيحة
- الأزرار تظهر/تختفي بناءً على الصلاحيات
- الأخطاء واضحة ومنطقية
- البيانات محمية من الوصول غير المصرح
- النظام لا يتعطل على أي خطأ

### ❌ اختبار فاشل إذا:
- موظف يرى صفحات لا يملك صلاحية الوصول إليها
- بيانات حساسة ظاهرة لموظف غير مصرح
- النظام يتعطل عند محاولة إجراء غير مسموح
- الصلاحيات لا تتحديث بعد تغيير الرتبة
